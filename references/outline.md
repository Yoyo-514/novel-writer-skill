# Lightweight Outline Repair and Normalization

Use this reference for quick outline cleanup, diagnosis, or normalization. For full blueprint creation, read `conception-blueprint.md`. For full module/volume outline design, read `outline-planning.md`. For chapter-level planning, read `chapter-outline.md`.

## Intake

Collect or infer:

- current blueprint anchor, if any
- current outline fragment
- target repair type: clarity, causality, pacing, module split, missing motivation, missing reveal, or chapter breakdown
- canon constraints and must-preserve elements

If the user provides a specialized outline prompt or topic guide, treat it as a high-priority project rule and preserve it separately rather than blending it invisibly into generic instructions.

## Outline Diagnostic Pass

Before rewriting, identify:

- missing motivation
- weak causal links
- repeated beats
- unresolved setup/payoff
- unclear chapter purpose
- character decisions that lack pressure
- timeline contradictions
- scenes that summarize when they should dramatize

Report the smallest useful set of changes. Apply the fix when the user asked for implementation.

## Revision Rules

- Preserve user canon unless explicitly asked to change it.
- Prefer causal repair over adding new lore.
- Split chapters by meaningful turns, not arbitrary word count.
- Add scenes only when they create pressure, reveal information, or change relationships.
- Mark inferred material clearly when the source outline is vague.
